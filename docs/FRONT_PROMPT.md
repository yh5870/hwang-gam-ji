────────────────────────────────────────────────────────
1) PRD 요약 (UI 기준)
────────────────────────────────────────────────────────
1. What: 가시거리 데이터를 중심으로 황령산의 조망 상태를 점수화하여 보여주는 앱.
2. Value: "오늘 황령산 가면 야경이 잘 보일까?"에 대한 직관적인 시각적 해답 제공.
3. JTBD: "퇴근 길에 스마트폰으로 황감 점수를 확인하고 드라이브 여부를 결정한다."
4. Primary Personas: 2030 카푸어/드라이브족, 야경 출사 작가, 부산 데이트 코스 설계자.
5. Non-Goals: 실제 기상청 API 연동 코드 작성, 회원가입 기능, 커뮤니티 서버 구축.

────────────────────────────────────────────────────────
2) 라우터 확정 (IA 고정)
────────────────────────────────────────────────────────
- "/" : 메인 대시보드 (현재 황감 점수 및 잭팟 알림)
- "/detail" : 가시거리, 미세먼지, 습도 등 상세 지표 카드뷰
- "/forecast" : 향후 24시간 예측 지수 리스트
- "/about" : 황감 지수 산출 공식 및 등급 가이드

────────────────────────────────────────────────────────
3) 공통 UI 규칙
────────────────────────────────────────────────────────

5) 라우터별 상세 명세
────────────────────────────────────────────────────────

[Route: "/"]
- 목적: 현재 조망 상태의 즉각적인 인지
- 레이아웃: 
  1. Top: 로고 및 일몰 카운트다운
  2. Center: 거대한 원형 게이지 + 애니메이션되는 점수 숫자 (0 -> Target)
  3. Bottom: "잭팟 발생" 배지 및 메인 상태 멘트
- 상태 머신:
  - initial: 스켈레톤 UI
  - success: 점수 카운팅 애니메이션 실행
  - jackpot: 화면 전체 파티클(폭죽) 효과 및 배경 그라데이션 변화
- ASCII Layout:
  [  Sunset: 01:24:05  ]
  [       ( 10,000 )      ]
  [    GOD-TIER VIEW!     ]
  [ [Detail] [Forecast]   ]

[Route: "/detail"]
- 목적: 가시거리 등 세부 수치 확인
- 레이아웃: 그리드 형태의 정보 카드 (가시거리, 미세먼지, 습도, 풍속)
- 표시 데이터: 가시거리는 'km' 단위로 가장 크게 표시 (삼성 날씨 스타일)

────────────────────────────────────────────────────────
6) 핵심 사용자 플로우
────────────────────────────────────────────────────────
1. 접속 플로우: "/" 진입 -> 점수 0에서 10,000으로 촤르륵 상승 -> 잭팟 이펙트 감상
2. 상세 확인: "/" 하단 [상세보기] 클릭 -> "/detail" 이동 -> 지표 확인
3. 정보 습득: 하단 탭바 [가이드] 클릭 -> "/about" 이동 -> 70km 기준 확인
